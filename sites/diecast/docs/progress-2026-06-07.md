# GrailPulse Die Cast Progress

This document records the GrailPulse Die Cast vertical as a GrailPulse collector product.

## Current Baseline

- Live domain: `https://diecast.grailpulse.com`
- Cloudflare Pages project: `grailpulse-diecast`
- App folder in this repository: `sites/diecast`
- KB path: `src/data/diecast-kb.json`
- Current KB: 11,724 records, 126 casting families, 10 brands
- Pricing mode: seed values
- Health gate: `npm test`

## Expansion Sync

The KB was expanded to 11,724 records and validated as a GrailPulse Die Cast app dataset.

The validation gate checks:

- at least 10% growth over the previous deployed baseline
- duplicate-free `diecast_id` values
- stable identity fields
- required field completeness
- condition-pricing coverage for every record
- family and photo-family minimums

## Build And Deploy

Standard verification:

```powershell
npm test
npm run build
npm run deploy:prep
```

Standard Cloudflare Pages upload:

```powershell
npx wrangler pages deploy out --project-name=grailpulse-diecast
```

`npm run deploy:prep` prunes generated App Router text payloads from `out/` so the expanded static HTML build stays under the current Cloudflare Pages file-count limit.

## Notes

- Generated folders such as `.next/`, `out/`, `.wrangler/`, `node_modules/`, and Playwright output are intentionally excluded from this backup.
- The app is family-first: browse and SEO should lead with casting families, then variant detail pages.
- Seed values are guide estimates and must not be presented as guaranteed current market value.
